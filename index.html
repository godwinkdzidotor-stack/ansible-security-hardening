SSH & Remote Access
Access Control
  • Disable root login over SSH.
  • Force key-based authentication where appropriate.
  • Tune SSH configuration for stronger ciphers and MACs.
ssh_config Least Privilege
Firewall & Services
Network Exposure
  • Baseline inbound/outbound rules with host firewall.
  • Stop and disable unnecessary network services.
  • Document open ports and justifications.
FirewallD / UFW / iptables
Password & Account Policy
Identity
  • Configure password complexity and aging.
  • Use PAM to limit failed login attempts.
  • Harden default account and lockout behaviors.
Playbook Layout
Best Practices
  • Roles segmented by concern (ssh, firewall, accounts, packages).
  • Variables stored separately for easy tuning by environment.
  • Idempotent tasks to support repeated runs safely.

Running the Playbooks
Usage
  1. Clone the repository to an Ansible control node.
  2. Adjust group vars and inventory to match your environment.
  3. Run ansible-playbook site.yml -i inventory.
  4. Review Ansible output and verify server state.
